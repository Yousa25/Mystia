今天刘铁猛老师讲的这部分内容是 C# 面向对象编程（OOP）的灵魂！特别是“引用变量与实例”的关系，以及“静态成员与实例成员”的区别。很多初学者在这里容易绕晕，但只要把它们代入到《夜雀食堂》的设定中，就会变得非常直观。


### 案例一：“气球与小孩”——引用变量与实例的奥秘
**学习目标**：深刻理解 `new` 关键字的作用，掌握引用变量（Reference Variable）和实例（Instance）的区别。
**世界观设定**：在幻想乡里，“妖怪”是一个抽象的概念（类），而“米斯蒂娅”是一个真实存在的个体（实例）。你不能直接对“妖怪”这个概念进行操作，只能对具体的某只妖怪操作。

**练习任务**：
1. **错误示范（两个气球）**：写一个控制台程序，尝试用 `(new Youkai()).Name = "幽幽子";` 然后下一行用 `(new Youkai()).Show();`。观察为什么名字设置不上去。（体会刘老师说的：每次 `new` 都会创建一个全新的、独立的内存实体）。
2. **正确示范（牵住气球）**：声明一个引用变量 `Customer vipGuest;`，然后用 `vipGuest = new Customer();` 进行赋值。接着通过 `vipGuest.Name = "西行寺幽幽子";` 来修改状态，最后调用方法展示她的信息。
3. **进阶体验（两个孩子牵同一个气球）**：创建另一个引用变量 `Customer anotherRef = vipGuest;`，修改 `anotherRef.Name = "贪吃的亡灵"`，然后再打印 `vipGuest.Name`。观察它们是否指向了同一个人！（这就是对象引用的本质）。

### 案例二：类的三大成员——构建“菜品模型”
**学习目标**：掌握属性（Property）、方法（Method）和事件（Event）的定义与作用。
**世界观设定**：夜雀食堂的招牌菜“烤八目鳗”。它不仅有自身的状态，还能被烹饪，甚至能在出锅时发出通知。

**练习任务**：
1. **定义属性（状态）**：创建一个 `Dish` 类，包含 `Name`（名称）、`Price`（价格）和 `IsReady`（是否做好）这三个属性。
2. **定义方法（行为）**：给 `Dish` 类添加一个 `Cook()` 方法。在这个方法里，将 `IsReady` 设置为 `true`，并在控制台打印出“滋滋作响的烤八目鳗出炉了！”
3. **定义事件（通知机制）**：这是 C# 独有的精华！在 `Dish` 类中声明一个事件 `public event Action OnCookingCompleted;`。当 `Cook()` 方法执行完毕时，触发这个事件。
4. **主程序测试**：在 `Main` 方法中实例化这道菜，并用 `+=` 订阅这个事件（比如订阅一个 lambda 表达式 `dish.OnCookingCompleted += () => Console.WriteLine("服务员幽谷响子：收到！马上端给客人！");`）。调用 `Cook()` 方法，观察事件的联动效果。

### 案例三：静态成员 vs 实例成员——全局规则与个人偏好
**学习目标**：区分 `static` 成员（属于类本身，全局共享）和非静态成员（属于具体对象，各自独立）。
**世界观设定**：夜雀食堂有“全店统一的营业时间”，同时每个进店的客人都有自己“独立的钱包余额”。

**练习任务**：
1. **定义静态成员**：在 `Izakaya`（居酒屋）类中，定义一个静态字段 `public static string OpeningHours = "18:00 - 12:00";`。这代表无论有多少个分店，或者不管多少个客人，这个规则是全局唯一的。可以直接通过 `Izakaya.OpeningHours` 访问。
2. **定义实例成员**：在 `Customer` 类中，定义一个非静态属性 `public int WalletBalance { get; set; }`。这代表每个客人的钱是各管各的。
3. **对比测试**：在 `Main` 方法中创建两个顾客实例 `customerA` 和 `customerB`。给 A 充值 1000 铜币，给 B 充值 500 铜币。分别打印他们的余额，证明实例成员是相互独立的。然后再打印一次 `Izakaya.OpeningHours`，体会静态成员的“类级别”特性。

###  学习小贴士
- 关于**事件（Event）**：刘老师特别强调它是 C# 的特色。你可以把它想象成食堂里的“传菜铃”。厨师做完菜按一下铃（触发事件），外面的服务员听到铃声就去上菜（响应事件）。这种“解耦”的思想是你以后学 WPF 按钮点击（Click 就是一个事件）的核心基础！
