# 类型、变量与对象（东方夜雀食堂版）

## 什么是类型

- 又名数据类型（Data Type）
  - 数据在幻想乡内存中存储时的“菜品型号”
  - 小盘子装大份量料理会溢出丢失食材、发生翻车错误
  - 大盘子装小份饭团会导致摊位空间浪费
  - 编程语言的数据类型与数学上的数值类型不完全相同，比如C#里的`string`不是数学概念里的字符串

## 类型在C#语言中的作用

- 一个C#类型中所包含的信息有：
  - 存储此类型变量所需的摊位内存空间大小
  - 此类型的值可表示的最大、最小范围（如客人满意度上限、金币余额下限）
  - 此类型所包含的成员（如方法、属性、事件等）：用`GetProperties()`和`GetMethods()`、`GetEvents()`可获取该类的属性、方法和事件
    ```csharp
    // 查看米斯蒂娅（Mistya）这个类型能做什么菜、有什么技能
    Type mistyaType = typeof(Mistya);
    PropertyInfo[] properties = mistyaType.GetProperties();
    MethodInfo[] methodInfos = mistyaType.GetMethods();

    foreach (PropertyInfo property in properties)
    {
        Console.WriteLine(property.Name); // 输出：CookingLevel, Stamina...
    }
    foreach (MethodInfo method in methodInfos)
    {
        Console.WriteLine(method.Name);   // 输出：CookRiceBall(), ServeDish()...
    }
    ```
  - 此类型由何基类派生而来（如`Mistya`继承自`NightChef`，`NightChef`继承自`Character`）
  - 程序运行的时候，此类型的变量分配在内存的什么位置
    - Stack简介（栈：摊位操作台，临时放调料的地方）
    - Stack overflow（栈溢出：操作台堆太多碗碟，直接塌了！）
    - Heap简介（堆：后厨仓库，存放大量食材和成品菜肴）
    - 使用Performance Monitor查看进程的堆内存使用量
    - 关于内存泄漏（忘了清理仓库，食材过期堆积如山，占用空间却不回收）

    函数调用用的栈，实例变量用堆。栈比较小但速度快，会发现栈溢出；堆很大，如果不回收会发生内存泄漏，导致内存浪费。
    使用Performance Monitor性能监视器可以看程序占用的内存情况。
    在进程（process）选项里找内存（private Bytes），观察你的食堂到底囤了多少没用的剩菜。
  - 此类型所允许的操作（运算）
    比如整数除法不会出现小数——就像你卖3个饭团赚90元，平均每个赚多少？如果用整数除法，结果是30而不是30.0，小数部分直接被切掉喂给灵梦了。

## C#语言的类型系统

- C#的五大数据类型
  - 类（Classes）：如`Form`、`Console`、`String`、`Mistya`、`Yuyuko`
    可使用`typeof()`运算符查看类型
  - 结构体（Structures）：如`Int32`、`Int64`、`Single`、`Double`、`IngredientCount`
  - 枚举（Enumerations）：如`HorizontalAlignment`、`Visibility`、`DishTag`（和风/中华/洋食）
  - 接口（Interfaces）：如`ICookable`、`IServeable`
  - 委托（Delegates）：如`OnGuestArrived`、`OnDishCompleted`

## 变量、对象与内存

- 什么是变量：变量=以变量名所对应的内存地址为起点、以其数据类型所要求的存储空间为长度的一块内存区域。
  - 从表面上来看，变量的用途是存储数据
  - 实际上，**变量表示了存储位置，并且每个变量都有一个类型，以决定什么样的值能够存入变量**
  - 变量一共有7种
    - 静态变量（如全服排行榜分数）、实例变量（成员变量/字段，如某位客人的好感度）、数组元素（菜单列表中的第3道菜）、值参数（传入函数的食材数量）、引用参数（ref，把整锅汤传进去改味道）、输出参数（out，做完菜返回评价等级）、局部变量（函数体里临时声明的变量）
  - 狭义的变量指局部变量，因为其他种类的变量都有自己的约定名称
    - 简单地讲，局部变量就是方法体（函数体）里声明的变量
  - 变量的声明
    - 有效的修饰符组合 类型 变量名 初始化器

  ![alt text](image-5.png)
  用excel表格可以表示变量在内存的结构：黄色部分代表操作系统使用的内存，不许用户使用。

  - 值类型的变量
    - 值类型没有独立的实例，所谓的“实例”与变量合二为一
    - 比如`int riceCount = 5;`，数字5就直接躺在栈上riceCount这块内存里，不需要额外去堆里找地方
  - 引用类型的变量与实例
    - 引用类型变量与实例的关系：引用类型变量里存储的数据是对象的内存地址
    - 引用类型：**先分给4字节的内存**（32位系统）或8字节（64位系统），初始都设置为0（null）。当使用`new`实例化后，从堆内存创建实例，**再找一块空内存，大小为成员变量中的值类型的大小**，把内存编号转化为二进制赋值给第一次分配的4字节内存中，给值类型的变量存储数据。
    - 比如`Mistya chef = new Mistya();`，栈上的`chef`只存了一个地址（指向堆里那个真正的米斯蒂娅对象），而她的名字、体力、烹饪等级等字段全都住在堆里。
  - 局部变量分配的是栈上的内存
  - 变量的默认值（值类型默认为0/false/null，引用类型默认为null）
  - 常量（值不可改变的变量，如`const int MAX_GUESTS = 10;`，一旦设定就不能再改了，就像幽幽子的食量永远填不满一样）
  - 装箱与拆箱（Boxing & Unboxing）
    - 装箱：把值类型包装成引用类型，比如把`int`放进`object`盒子里，需要在堆上开辟新空间
    - 拆箱：把盒子里的值类型取出来，需要类型转换验证，否则可能抛出异常（就像你以为盒子里是饭团，结果打开发现是灵梦的塞钱箱……）
