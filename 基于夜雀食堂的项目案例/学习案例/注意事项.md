这份案例专门为你练习 **表单（Form）与交互元素** 设计。代码里我特意加上了详细的中文注释，帮你理清每个标签的作用。

你可以直接复制到编辑器里运行，看看浏览器最原始的表单长什么样：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>幻想乡大图书馆·入馆申请表</title>
</head>
<body>
    <!-- h1 定义页面主标题 -->
    <h1>📚 幻想乡大图书馆·入馆申请表</h1>
    <p><em>温馨提示：未经帕秋莉大人允许，严禁在馆内使用火系魔法！</em></p>
    
    <hr>

    <!-- 
      form 标签：整个表单的容器
      action="#"：提交后跳转的地址（目前没写后端，先留空或#）
      method="post"：数据传输方式（涉及隐私信息通常用 post）
    -->
    <form action="#" method="post">
        
        <!-- 
          fieldset + legend：将表单内容分组并添加标题
          这是语义化表单的最佳实践，能让结构更清晰
        -->
        <fieldset>
            <legend>【基础信息申报】</legend>
            
            <!-- label + input：文本输入框 -->
            <!-- for 属性的值必须和 input 的 id 一致，点击文字也能聚焦输入框 -->
            <label for="name">姓名/称号：</label><br>
            <input type="text" id="name" placeholder="例如：博丽灵梦" required><br><br>
            
            <!-- select：下拉选择菜单 -->
            <label for="race">种族：</label><br>
            <select id="race">
                <option value="" disabled selected>-- 请选择您的种族 --</option>
                <option value="human">人类</option>
                <option value="youkai">妖怪</option>
                <option value="fairy">妖精</option>
                <option value="half">半妖/其他</option>
            </select><br><br>
            
            <!-- radio：单选按钮 -->
            <!-- 注意：name 属性必须相同，才能实现“多选一”的效果 -->
            <label>危险等级评估：</label><br>
            <input type="radio" id="level-low" name="danger-level" value="low">
            <label for="level-low">低（仅看书喝茶）</label><br>
            <input type="radio" id="level-mid" name="danger-level" value="mid">
            <label for="level-mid">中（偶尔会放符卡）</label><br>
            <input type="radio" id="level-high" name="danger-level" value="high">
            <label for="level-high">高（可能拆了图书馆）</label><br><br>
        </fieldset>

        <br>

        <fieldset>
            <legend>【能力与申请理由】</legend>
            
            <!-- checkbox：复选框 -->
            <label>已阅读并同意以下条款：</label><br>
            <input type="checkbox" id="rule1" required>
            <label for="rule1">禁止在馆内吃烤鸟蛋</label><br>
            <input type="checkbox" id="rule2" required>
            <label for="rule2">禁止对书籍使用物理攻击</label><br><br>
            
            <!-- textarea：多行文本输入框 -->
            <label for="reason">入馆申请理由：</label><br>
            <textarea id="reason" rows="5" cols="40" placeholder="请详细说明您为什么需要进入大图书馆..."></textarea><br><br>
        </fieldset>

        <!-- button：提交按钮 -->
        <!-- type="submit" 表示点击后会触发表单提交动作 -->
        <button type="submit">向帕秋莉提交申请</button>
        
        <!-- reset 按钮：清空所有填写的内容 -->
        <button type="reset">重新填写</button>
    </form>
</body>
</html>
```

### 💡 敲代码时的核心关注点：

1. **`label` 的 `for` 绑定**：一定要和对应 `input/select/textarea` 的 `id` 对上！这是无障碍设计的基础，也是前端规范。
2. **单选按钮的 `name`**：所有属于同一组的 `radio`，`name` 必须完全一样，否则就能同时选中多个，失去单选的意义。
3. **`fieldset` 分组**：当表单很长时，用它来划分区块，不仅视觉上清晰，屏幕阅读器也能更好地解析网页结构。
4. **`required` 属性**：加上这个单词，如果用户不填这项就点提交，浏览器会自动弹出提示拦截，这是纯 HTML 就能实现的简单验证！

<!DOCTYPE html>

这份案例专门为你练习 **表格（Table）与结构化数据** 设计。

在网页中，表格是用来展示二维数据的最佳工具。为了让代码结构更清晰、对屏幕阅读器更友好，我们不仅要使用 `<table>`、`<tr>`、`<td>`，还要引入语义化标签 `<thead>`（表头）、`<tbody>`（主体）和 `<tfoot>`（表尾），以及用于合并单元格的 `colspan` 属性。

你可以直接复制到编辑器里运行，观察浏览器是如何自动为表格添加网格线的：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>博丽神社·参拜者名录</title>
</head>
<body>
    <!-- h1 定义页面主标题 -->
    <h1>⛩️ 博丽神社 · 今日参拜者名录</h1>
    <p><em>记录幻想乡居民的日常打卡情况</em></p>
    
    <hr>

    <!-- 
      table 标签：整个表格的容器
    -->
    <table border="1" cellpadding="5" cellspacing="0">
        
        <!-- thead：表头区域，通常包含列标题 -->
        <thead>
            <tr>
                <th>姓名</th>
                <th>种族</th>
                <th>赛钱金额</th>
                <th>备注</th>
            </tr>
        </thead>

        <!-- tbody：表体部分，包含主要的数据行 -->
        <tbody>
            <tr>
                <td>雾雨魔理沙</td>
                <td>人类</td>
                <td>1枚铜币</td>
                <td>借走了《魔法基础入门》，未归还</td>
            </tr>
            <tr>
                <td>琪露诺</td>
                <td>妖精</td>
                <td>冰块 x 3</td>
                <td>试图用冰块支付被灵梦拒绝</td>
            </tr>
            <tr>
                <td>八云紫</td>
                <td>妖怪</td>
                <td>黄金 1kg</td>
                <td>神出鬼没，突然出现又消失</td>
            </tr>
        </tbody>

        <!-- tfoot：表尾部分，常用于统计信息或合计行 -->
        <tfoot>
            <tr>
                <!-- colspan="4" 表示这个单元格横跨4列 -->
                <td colspan="4"><strong>今日总计：</strong> 3位访客 | 实际赛钱收入：极低</td>

</body>
</html>
```

### 💡 敲代码时的核心关注点：

1. **语义化结构**：注意看 `<thead>`、`<tbody>` 和 `<tfoot>` 的位置。虽然省略它们也能显示表格，但加上它们能让你的 HTML 结构极其规范，这也是现代前端开发的最佳实践。
2. **表头标签 `<th>`**：表头一定要用 `<th>` (table header) 而不是 `<td>`。浏览器默认会把 `<th>` 里的文字加粗并居中，以区别于普通数据。
3. **单元格合并 `colspan`**：在最底部的“总计”行，因为我们只需要一个贯穿整行的说明文字，所以使用了 `colspan="4"`。这个数字代表它要跨越的列数，刚好等于上面定义的4个表头列。
4. **原生样式属性**：为了让你在没有 CSS 的情况下也能看清表格边框，我加了 `border="1"` 和 `cellpadding="5"`。*（注：在实际的现代开发中，我们会去掉这些 HTML 属性，改用 CSS 来控制边框和间距，这里只是为了纯 HTML 练手方便观察。）*

这份案例专门为你练习 **语义化标签（Semantic Tags）与内容组织** 设计。

在网页中，`<div>` 和 `<span>` 只是毫无意义的盒子，而 HTML5 引入的语义化标签（如 `<article>`, `<section>`, `<header>`, `<aside>` 等）则自带“含义”。它们能让搜索引擎和屏幕阅读器瞬间看懂你的网页结构。

你可以直接复制到编辑器里运行，观察浏览器是如何自动排列这些带有明显区块感的元素的：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>东方角色百科·人物档案</title>
</head>
<body>
    <!-- 
      article 标签：定义独立的内容块。
      这里的整篇人物档案就是一个完整的、可以独立分发的文章。
    -->
    <article>
        
        <!-- header 标签：通常包含标题或引导性内容 -->
        <header>
            <h1>🌸 博丽灵梦 (Hakurei Reimu)</h1>
            <p><strong>称号：</strong>乐园的巫女 | <em>种族：人类</em></p>
        </header>

        <hr>

        <!-- section 标签：定义文档中的一个通用区块/章节 -->
        <section>
            <h2>【基础设定】</h2>
            <p>
                博丽神社的巫女，负责解决幻想乡内的异变。虽然平时总是抱怨没有香火钱，
                但在关键时刻总能展现出压倒性的实力。拥有在空中飞翔程度的能力。
            </p>
        </section>

        <!-- aside 标签：定义与主内容相关但可独立存在的侧边信息 -->
        <aside>
            <h3>📊 角色数据面板</h3>
            <ul>
                <li>攻击：★★★★☆</li>
                <li>防御：★★☆☆☆</li>
                <li>财力：★☆☆☆☆ (极度贫穷)</li>
                <li>爱好：喝茶、退治妖怪</li>
            </ul>
        </aside>

        <hr>

        <!-- section 标签：继续划分新的章节 -->
        <section>
            <h2>【人际关系网】</h2>
            <dl>
                <!-- dl (description list)：非常适合做这种名词解释式的列表 -->
                <dt>雾雨魔理沙</dt>
                <dd>经常来神社串门的人类魔法使，也是灵梦最常一起退治异变的搭档。</dd>
                
                <dt>八云紫</dt>
                <dd>神出鬼没的大妖怪，经常在背后暗中观察并给灵梦派活。</dd>
            </dl>
        </section>

        <!-- footer 标签：定义文章的尾部信息，如版权、作者、参考资料等 -->
        <footer>
            <p><small>
                📝 资料来源：《东方求闻史纪》 | 
                ⚠️ 本词条由野生程序员整理，如有错漏请在评论区指正。
            </small></p>
        </footer>

    </article>
</body>
</html>
```

### 💡 敲代码时的核心关注点：

1. **告别万能 `<div>`**：试着理解每个标签的“潜台词”。`<article>` 代表“这是一篇完整文章”，`<section>` 代表“这是文章的一个段落/章节”，`<aside>` 代表“这是补充说明的侧边栏”。用对标签，你的代码会显得非常专业。
2. **认识 `<dl>` 描述列表**：在这个案例的人际关系网中，我用到了 `<dl>` (definition list)、`<dt>` (term) 和 `<dd>` (description)。这组标签在做“名词+解释”、“属性+数值”这种成对出现的数据时，比 `<ul>` 好用得多！
3. **内容的模块化**：你会发现整个页面被清晰地切割成了头部、正文几个部分、补充信息和尾部。这就是前端开发中常说的“组件化思维”的雏形——以后学 CSS 的时候，你可以很方便地单独给 `<aside>` 加上浮动或背景色，而不影响其他部分。

把这个代码手敲一遍，试着再增加一个 `<section>`，写一写灵梦的“招牌符卡”，感受一下语义化标签带来的清晰结构！

1. **锚点跳转原理**：注意看 `<a href="#mountain-intro">` 和 `<section id="mountain-intro">` 之间的呼应。`#` 符号告诉浏览器在当前页面内部寻找匹配 `id` 的元素并滚动过去。这是所有长网页（如维基百科、说明书）必备的基础功能。
2. **专属导航标签 `<nav>`**：它和 `<header>`、`<footer>` 一样，属于语义化标签。把导航链接放进 `<nav>` 里，屏幕阅读器就能立刻知道：“哦，这是用来跳转的菜单”。
3. **结构化的层级**：整个页面遵循了 `nav (导航) -> header (页头) -> section (各个独立章节) -> footer (页脚)` 的经典文档流布局。以后当你学习 CSS 时，这种清晰的骨架能让你极其轻松地给每个区块排版。

### 💡 核心知识点复盘：

1. **标题标签 (`<h1>` - `<h6>`)**：
   * `<h1>` 是老大，通常一个页面只有一个，代表网页的核心主题。
   * `<h2>` 到 `<h6>` 依次变小，用来划分内容的层级结构。它们自带字号大小和上下间距。
2. **段落标签 (`<p>`)**：
   * 专门用来包裹一段完整的文字。它是块级元素，前后会自动产生一定的空白间距。
3. **换行标签 (`<br/>`)**：
   * 它只是一个“回车键”。比如我们在价格和库存之间用 `<br/>` 换行，因为它们同属于“烤鸡肉串”这一个段落的信息。注意不要滥用它来做段落间的分隔！
4. **文本格式化标签**：
   * `<strong>`：语义上的“重要”，视觉上通常是**加粗**。
   * `<em>`：语义上的“强调/语气”，视觉上通常是*斜体*。
   * `<del>`：表示被废弃或删除的内容，视觉上会加上~~删除线~~。
5. **div标签**：
   * 它是没有任何语义的“透明收纳盒”。在这里我们用三个 `<div>` 分别把页头、内容、页脚装起来，方便以后写 CSS 时进行整体布局控制。
